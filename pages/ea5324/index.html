在博客上发布一篇文章的完整流程！

提示

本文会介绍发布一篇文章的具体流程。

# 0. 序言

本文的大致会介绍下面的内容：

  • 一篇文章需要有哪些配置（front matter (opens new window)）；
  • 我们需要为这篇博客配置路由（即这篇文章会出现博客的哪个地方）;
  • 我们需要部署到指定网址，以让别人看到这边文章。

# 1. 文章的基础配置

两个---之间的内容是就是这篇文章的 front matter 配置部分，下面对其基本属性进行介绍:

  • title: 这篇文章的标题;
  • date: 发布时间;
  • permalink: 永久链接，用于跳转; ⭐️
  • categories: 分类;
  • tags: 标签;
  • author: 作者信息。

permalink 这个属性我们后面还会再看到，这代表了这篇文章的 唯一永久 链接。

如果你觉得上面的描述不够清晰，请参照 官方大佬 (opens new window) 的文档。

# 2. 路由设置

我们要实现的效果是，点击右上角的导航栏 博客，然后会跳转到一个 目录页，该 目录页会显示该分类下的所有文章。效果大致是这丫子：

可以看到，我们需要有两个配置。第一个是在导航栏那里弄一个博客的东西，第二个弄一个目录页，下面请看2.1和2.2的操作。

# 2.1 配置链接

在 config.ts 目录中配置对应的路由链接，内容如下：

themeConfig: {
// 导航配置
nav: [
{ text: '首页', link: '/' },
{ text: '博客', link: '/note/blog/' },
]
}

# 3.部署

部署比较简单，创建一个github仓库，然后执行下面的脚本即可：

#!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e


push_addr="https://github.com/ShuaiLi-XY/blog.git" # git提交地址，也可以手动设置，比如：push_addr=git@github.com:xugaoyi/vuepress-theme-vdoing.git
commit_info="git describe --all --always --long"
dist_path=docs/.vuepress/dist # 打包生成的文件夹路径
push_branch=gh-pages # 推送的分支

# 生成静态文件
yarn build:win

# 进入生成的文件夹
cd $dist_path

git init
git add -A
git commit -m "deploy, $commit_info"
git push -f $push_addr HEAD:$push_branch

cd -
rm -rf $dist_path
